Background

Here's some behind-the-scenes on this website that you are on right now! I wanted to code a portfolio website from scratch using HTML, CSS and Javascript. That way, I could learn the basics of each programming language and get a better understanding of what goes on in website design.

Research

To decide what the website would look and feel like, I studied various portfolios from UX designers. I also studied miscellaneous websites like news websites and blogs to see how they handled their navigation.

I made some rough wireframes which you can see a sample of here.

I also learned HTML, CSS and Javascript with no essentially no prior knowledge of front-end programming languages. It was an excellent learning experience.

Communication

I wanted to make my site tactile and playful.

Why tactile and playful? Well, I wanted to signify to the user what was clickable and what wasn’t, and I wanted to make the website fun to navigate. To accomplish these goals, I made big, colorful buttons with hover effects and drop shadows. I drew funny custom illustrations. I used an un-serious looking font, but made sure to preserve information hierarchy using bolding, font sizes, and underlining.

One of the key takeaways I got from building the site is that trying to program can get in the way of thinking about the user experience. That is, I could have improved the UX for some features, but it would have been too hard to program.

I think learning web programming gives me more appreciation for developers and also helps me figure out what some limitations to website design are, but it’s probably better to specialize in one or the other rather than dabbling in both. Also, as mentioned before, I've had no formal training in visual design, so this would again be a good reason for me to pursue an HCI degree and fill in that knowledge gap.
